Знакомьтесь с httpx

Когда вы начинаете работать с `HTTP-запросами` в `Python`, одна из самых популярных библиотек — это `Requests`. Но если вы хотите немного улучшить свой опыт, есть отличная альтернатива: `httpx`. В этой статье мы познакомим вас с `httpx`, расскажем о его преимуществах и объясним, чем он лучше `Requests`. Кроме того, приведем множество примеров, чтобы вы могли легко освоить использование этой библиотеки.

## Что такое httpx? ❓
`httpx` — это современная и мощная библиотека для отправки `HTTP-запросов` в `Python`. Она является более гибкой и функциональной, чем `Requests`, и предоставляет дополнительные возможности, такие как асинхронные запросы.

## Почему стоит использовать httpx? 🌟
### 1.Асинхронная работа 🔥
Одна из главных особенностей `httpx` — поддержка асинхронных запросов. Это означает, что вы можете отправлять несколько запросов одновременно, не блокируя выполнение программы. Это делает `httpx` отличным выбором для приложений, где важна высокая производительность.

#### Пример асинхронного запроса с httpx:

```python
import httpx
import asyncio

async def fetch_data():
    async with httpx.AsyncClient() as client:
        response = await client.get('https://jsonplaceholder.typicode.com/posts')
        print(response.text)

asyncio.run(fetch_data())
```
В этом примере мы используем асинхронный клиент `httpx` для выполнения запроса без блокировки программы.

### 2. Поддержка HTTP/2 🔝
`httpx` поддерживает `HTTP/2`, что может значительно улучшить производительность при работе с сервером, который поддерживает эту версию протокола. `HTTP/2` позволяет уменьшить задержки и ускорить передачу данных.

### 3. Подключение через прокси и поддержку тайм-аутов 🔒
`httpx` имеет встроенную поддержку прокси-серверов и более гибкую настройку тайм-аутов для запросов, что позволяет легко настроить взаимодействие с различными внешними сервисами.
### 4. Поддержка перенаправлений 🔁
`httpx` обрабатывает редиректы (перенаправления) запросов, что позволяет автоматически следовать за адресами, если сервер отправляет перенаправление.

### 5. Совместимость с Requests 🔄
`httpx` поддерживает большинство функций, которые есть в `Requests`. Если вы уже знакомы с `Requests`, вам будет легко перейти на `httpx`, так как синтаксис почти идентичен.

## Чем httpx лучше Requests? 🚀
- Асинхронность: как мы уже упомянули, `httpx` поддерживает асинхронные запросы, что позволяет делать несколько запросов одновременно. В `Requests` такого функционала нет.

- Производительность: благодаря поддержке `HTTP/2` и асинхронной работе, `httpx` может быть быстрее и более производительным для обработки большого количества запросов.

- Современные функции: httpx предоставляет дополнительные возможности, такие как поддержка прокси-серверов, гибкие тайм-ауты и работу с более новыми стандартами.

### Примеры использования httpx 🛠️
#### Пример 1: Отправка простого GET-запроса
Этот пример показывает, как отправить простой `GET-запрос` и получить данные с веб-сайта.

```python
import httpx

# Отправляем GET-запрос на сайт
response = httpx.get('https://jsonplaceholder.typicode.com/posts')

# Выводим текст ответа
print(response.text)
```
Здесь мы используем `httpx.get`, чтобы получить данные с ресурса. Важно заметить, что возвращаемый объект `response` имеет такие атрибуты, как `text`, `status_code`, `headers`, которые позволяют работать с данными ответа.

#### Пример 2: Отправка POST-запроса
Для отправки данных на сервер используем `POST-запрос`. В этом примере мы отправляем данные в формате `JSON`.

```python
import httpx

# Данные для отправки
data = {
    "title": "foo",
    "body": "bar",
    "userId": 1
}

# Отправляем POST-запрос
response = httpx.post('https://jsonplaceholder.typicode.com/posts', json=data)

# Выводим статус и полученные данные
print(f"Статус: {response.status_code}")
print(f"Ответ: {response.json()}")
```
В этом примере мы используем httpx.post для отправки данных на сервер в формате `JSON`. Ответ сервера можно получить через метод response.`json`(), который автоматически парсит `JSON-ответ`.

#### Пример 3: Использование асинхронных запросов с httpx.AsyncClient
Асинхронность — это одна из главных особенностей `httpx`. Рассмотрим пример, как отправить несколько запросов одновременно.

```python
import httpx
import asyncio

async def fetch_data(url):
    async with httpx.AsyncClient() as client:
        response = await client.get(url)
        return response.text

async def main():
    urls = [
        'https://jsonplaceholder.typicode.com/posts/1',
        'https://jsonplaceholder.typicode.com/posts/2',
        'https://jsonplaceholder.typicode.com/posts/3'
    ]
    tasks = [fetch_data(url) for url in urls]
    results = await asyncio.gather(*tasks)

    # Выводим первые 100 символов каждого ответа
    for result in results:
        print(result[:100])

# Запуск асинхронных запросов
asyncio.run(main())
```
В этом примере мы создаем несколько асинхронных запросов и выполняем их одновременно с помощью `asyncio.gather`. Это позволяет нам сэкономить время, отправляя несколько запросов параллельно.

#### Пример 4: Обработка ошибок
Иногда запрос может завершиться ошибкой, например, если сервер не найден. Мы можем обработать такие ошибки с помощью конструкции `try/except`.
```python
import httpx

try:
    # Отправляем запрос на несуществующий сайт
    response = httpx.get('https://jsonplaceholder.typicode.com/nonexistent')
    response.raise_for_status()  # Если статус не 200, выбрасывается исключение
    print(response.text)
except httpx.HTTPStatusError as e:
    print(f"Ошибка HTTP: {e.response.status_code}")
except httpx.RequestError as e:
    print(f"Ошибка запроса: {e}")
```
Здесь мы обрабатываем ошибки `HTTP` и запроса, используя исключения `httpx`.`HTTPStatusError` и `httpx`.`RequestError`. Это позволяет точно определить, что пошло не так при выполнении запроса.

#### Пример 5: Использование прокси с httpx
`httpx` поддерживает использование прокси-серверов, что может быть полезно для обхода ограничений или скрытия реального `IP-адреса`.

```python
import httpx

# Настроим прокси
proxies = {
    "http://": "http://your-proxy.com",
    "https://": "http://your-proxy.com"
}

# Отправляем запрос через прокси
with httpx.Client(proxies=proxies) as client:
    response = client.get('https://jsonplaceholder.typicode.com/posts')
    print(response.text)
```
В этом примере мы указываем прокси-сервер в параметре `proxies`, чтобы все запросы шли через этот сервер.
`
#### Пример 6: Установка тайм-аутов
Если сервер не отвечает в течение определенного времени, можно установить тайм-аут для запроса, чтобы избежать блокировки программы.

```python
import httpx

try:
    # Устанавливаем тайм-аут 5 секунд
    response = httpx.get('https://jsonplaceholder.typicode.com/posts', timeout=5)
    print(response.text)
except httpx.TimeoutException:
    print("Превышен тайм-аут!")
```
Здесь мы устанавливаем тайм-аут в 5 секунд. Если сервер не ответит в течение этого времени, будет выброшено исключение `httpx`.`TimeoutException`.

## Заключение 🌟
`httpx` — это мощная и современная библиотека для `HTTP-запросов`, которая имеет несколько ключевых преимуществ перед `Requests`. С поддержкой асинхронных запросов, `HTTP/2` и другими полезными фишками, `httpx` позволяет значительно улучшить производительность вашего кода, особенно когда вам нужно отправлять много запросов одновременно.
