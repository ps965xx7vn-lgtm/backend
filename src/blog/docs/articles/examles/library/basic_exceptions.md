# Базовые исключения в Python

Когда вы пишете код на `Python`, иногда что-то может пойти не так: программа может попытаться разделить на ноль, открыть несуществующий файл или обработать неправильный ввод пользователя. Такие ошибки называются исключениями (`exceptions`), и они могут вызвать сбой программы. Но не переживайте! `Python` предоставляет инструменты для обработки исключений, чтобы ваш код работал устойчиво и предсказуемо. 🌟

## Что такое исключение? ❓
Исключение — это ошибка, которая возникает во время выполнения программы. Если исключение не обработать, программа завершится аварийно. Однако с помощью конструкции `try/except` вы можете поймать ошибку и выполнить альтернативные действия.

## Как работает try/except? 🛠️
```python
try:
    # Код, который может вызвать исключение
    number = int(input("Введите число: "))
    result = 10 / number
    print("Результат:", result)
except ZeroDivisionError:
    # Обработка деления на ноль
    print("Ошибка: деление на ноль невозможно.")
except ValueError:
    # Обработка некорректного ввода (например, если введен текст вместо числа)
    print("Ошибка: введено не число.")
```

## Распространённые исключения 📋
- `ZeroDivisionError`: попытка деления на ноль.
```python
print(5 / 0)  # Ошибка: деление на ноль
```

- `ValueError`: неправильный тип данных.
```python
int("abc")  # Ошибка: невозможно преобразовать текст в число
```

- `IndexError`: выход за пределы списка.
```python
list = [1, 2, 3]
print(list[5])  # Ошибка: элемента с индексом 5 нет
```

- `FileNotFoundError`: файл не найден.
```python
with open("non_existing_file.txt") as f:
    content = f.read()  # Ошибка: файл не найден
```
## Зачем обрабатывать исключения? 🎯
Предотвращение сбоев: программа продолжит работу даже при ошибках.
Удобство для пользователя: показывать понятные сообщения об ошибках.
Безопасность: контролировать неожиданные ситуации.
