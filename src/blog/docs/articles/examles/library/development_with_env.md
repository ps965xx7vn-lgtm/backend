# Прячьте чувствительные данные: Введение в Environs
Когда вы пишете код, важно защитить конфиденциальную информацию, такую как пароли, ключи API или данные доступа к базам данных. Если хранить такие данные прямо в коде, это может привести к утечке информации, особенно если код попадёт в публичный репозиторий. Решение? Использовать переменные окружения! 🌍

## 🤔 Что такое чувствительные данные?
Это любая информация, которую нельзя делать публичной:
- Пароли от баз данных.
- Ключи доступа к внешним сервисам (например, к `Telegram API` или `GitHub`).
- Логины и другие учетные данные.

## 🗂️ Почему нельзя хранить их в коде?
Безопасность: Если код станет публичным, данные будут доступны всем.
Гибкость: Легче менять конфиденциальные данные в одном месте, чем переписывать код.
Лучшая практика: Важный стандарт в разработке программного обеспечения.
## 🚀 Environs — ваш помощник
Environs — это библиотека для Python, которая помогает работать с переменными окружения. Она загружает данные из файла .env и делает их доступными в коде.

## 📌 Установка Environs:
Сначала установите библиотеку:

```bash
pip install environs
```
## 📄 Создание .env файла:
Создайте файл с именем .env в корне проекта. В нём хранятся переменные окружения:

``` plaintext
API_KEY=1234567890abcdef
DB_PASSWORD=mysecretpassword
```
## 📝 Использование Environs в коде:
```python
Копировать код
from environs import Env

# Инициализируем environs
env = Env()
env.read_env()

# Получаем данные из .env
api_key = env("API_KEY")
db_password = env("DB_PASSWORD")

# Выводим данные (в реальном коде так делать не стоит!)
print("API Key:", api_key)
print("Database Password:", db_password)
```

## 🎯 Итог:
Хранение чувствительных данных в переменных окружения — важный шаг для безопасности вашего проекта. Environs упрощает этот процесс и делает код чище и безопаснее. Не храните секреты в коде — прячьте их правильно! 🔒🚀
