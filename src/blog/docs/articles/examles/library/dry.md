#DRY или как не повторяться: главный принцип чистого кода
Повторять один и тот же код — это как рассказывать одну и ту же историю по кругу: скучно, утомительно и легко запутаться. В программировании есть правило, которое спасает от этого: `DRY (Don't Repeat Yourself)`, что значит «не повторяйся». Давайте разберёмся, почему это важно и как применять этот принцип на практике!

## Что такое DRY и зачем он нужен?
Представьте, что у вас есть функция, которая считает сумму двух чисел. В одной части программы вы написали её, а через неделю забыли и сделали то же самое снова, но чуть иначе. Когда нужно исправить ошибку или добавить новую возможность, придётся менять обе версии. `DRY` призывает: одна идея — один кусочек кода.

Так вы сокращаете ошибки, упрощаете поддержку и экономите время.

Как избавиться от повторений?
### Пример с функциями и методами:
❌ Плохо: Дублирование кода:

```python
print("Добро пожаловать!")
# ...
print("Добро пожаловать!")  # То же самое снова
```
✔️ Хорошо: Используйте функцию:

```python
def print_welcome():
    print("Добро пожаловать!")
print_welcome()
print_welcome()
```

### Пример с циклами вместо копипаста:
❌ Плохо: Один и тот же код повторяется несколько раз:

```python
send_email("user1@example.com")
send_email("user2@example.com")
send_email("user3@example.com")
```
✔️ Хорошо: Цикл убирает дублирование:

```python
users = ["user1@example.com", "user2@example.com", "user3@example.com"]
for user in users:
    send_email(user)
```
### Пример с параметрами и аргументами:
❌ Плохо: Функции с похожей логикой повторяются:

```python
def greet_alice():
    print("Привет, Alice!")

def greet_bob():
    print("Привет, Bob!")
```
✔️ Хорошо: Одна функция для всех:

```python
def greet(name):
    print(f"Привет, {name}!")

greet("Alice")
greet("Bob")
```
## Как найти повторения?
- Обратите внимание на дублирующиеся строки кода.
- Проверьте, похожи ли блоки по структуре.
- Используйте линтеры (например, `Pylint`), которые подскажут, где вы повторяетесь.
- Исключения из правила `DRY`
- Иногда лучше «повторить» что-то для ясности. Например, если объединение кода усложнит его понимание, лучше оставить простую и понятную структуру.

## Заключение
Принцип `DRY`— ваш верный помощник на пути к чистому и понятному коду. Чем меньше дублирования, тем проще поддержка и развитие вашего проекта. Помните: не повторяйтесь, и ваш код станет намного лучше!
