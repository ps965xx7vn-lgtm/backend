# Понимаем if __name__ == "__main__" в Python
Когда вы начинаете изучать `Python`, вы можете встретить странную строчку кода:

```python
if __name__ == "__main__":
    # какой-то код
```
Что это значит? Почему она важна? Давайте разберёмся! 🤓✨

## Что такое __name__? 🧐
``__name__`` — это специальная переменная в `Python`. Она показывает, как был запущен файл:

Если файл запущен напрямую (например, через команду `python my_script.py`), то `__name__` будет равен `"__main__"`.
Если файл импортирован как модуль в другой программе, то `__name__` будет равен имени файла (без расширения .`py`).
## Простой пример 👇
Создадим файл `example.py:`

```python
def say_hello():
    print("Привет!")

if __name__ == "__main__":
    say_hello()
```
Теперь посмотрим, что произойдёт:

#### Запускаем файл напрямую:

```bash
python example.py
```
#### Вывод:

```
Привет!
```
#### Импортируем файл в другой программе:

```python
import example
```
#### Вывод:

```scss
(ничего не происходит)
```
## Зачем это нужно? 🤔
#### 1. Контроль точек входа:
Это позволяет писать код, который можно использовать и как самостоятельную программу, и как модуль для других программ.

#### 2. Избегание нежелательного выполнения:
Если у вас есть функции или тесты, вы можете быть уверены, что они выполнятся только при запуске файла напрямую.

## Реальный пример использования 🛠️
Представьте, что вы пишете калькулятор:

```python
# calculator.py
def add(a, b):
    return a + b

if __name__ == "__main__":
    print("Результат сложения:", add(5, 3))
```
Теперь вы можете:

Запустить калькулятор как самостоятельную программу.

Импортировать функцию `add()` в другом файле без запуска кода внутри `if __name__ == "__main__"`:

```python
from calculator import add
print(add(10, 2))
```
## Итог 🎯
`if __name__ == "__main__"` — это способ убедиться, что ваш код выполняется только тогда, когда вы запускаете файл напрямую. Если файл импортируется как модуль, этот блок игнорируется.

Это простой, но мощный инструмент для создания гибких и понятных программ! 🚀😊
