# Что такое статусы HTTP-ответов и как с ними работать в Requests
Когда вы отправляете запрос через библиотеку `Requests` в `Python`, сервер не просто молча отправляет данные. Он говорит с вами на специальном «языке» — языке статусов ответа. Это как дорожные знаки в интернете: они показывают, что всё идёт хорошо или что-то пошло не так. Давайте разберёмся, что это за статусы и как с ними работать!

## Что такое HTTP-статусы? 🤔
HTTP-статус — это код из трёх цифр, который сервер возвращает вместе с ответом на ваш запрос. Каждая цифра что-то значит:

- 1xx — Информационные сообщения (редко встретите).
- 2xx — Всё ОК! Сервер выполнил запрос.
- 3xx — Редирект: данные переехали или вас перенаправляют.
- 4xx — Ошибка клиента: что-то не так с вашим запросом.
- 5xx — Ошибка сервера: проблема на стороне сервера.


## Как проверить статус ответа в Requests? 📋

```python
import requests

url = 'https://jsonplaceholder.typicode.com/posts/1'
response = requests.get(url)

# Проверим статус
print(f'Статус-код: {response.status_code}')
```
Если всё прошло гладко, сервер вернёт 200 — это значит «OK! Запрос выполнен успешно».


Обычные статусы, которые стоит знать:
- 200 OK — Всё хорошо! Получили, что хотели.
- 201 Created — Успешно создан новый ресурс (например, пост или пользователь).
- 400 Bad Request — Ой! Что-то не так в вашем запросе.
- 401 Unauthorized — Доступ запрещён. Нужна авторизация.
- 404 Not Found — Ресурс не найден. Классика!
- 500 Internal Server Error — Сервер сломался или запутался.
