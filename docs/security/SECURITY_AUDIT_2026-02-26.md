## ❗ ГДЕ ПРОБЛЕМА?

### Вывод: Man-in-the-Middle атака на уровне провайдера в Грузии

Редирект на `/lander` происходит **ВНЕ вашего сервера**. Возможные причины:

1. **DNS Hijacking провайдера в Грузии** (наиболее вероятно)
   - Грузинский ISP подменяет DNS и перенаправляет трафик
   - Только пользователи из Грузии видят проблему

2. **HTTP Proxy/CDN провайдера**
   - Провайдер использует прозрачный прокси
   - Прокси подменяет контент для отдельных доменов

3. **Проблема на клиентской стороне** (менее вероятно)
   - Вредоносное браузерное расширение
   - Локальный hosts файл скомпрометирован
   - DNS cache poisoning на роутере

4. **DPI (Deep Packet Inspection) на уровне страны**
   - Государственная цензура/фильтрация
   - Подмена трафика для определенных доменов

## 🔧 Решение проблемы

### Немедленные действия:

1. **Проверить панель Timeweb**
   - Зайти в https://timeweb.com → DNS управление
   - Убедиться что A-запись `pylandschool.com` = `72.56.105.54`
   - Проверить нет ли дополнительных записей

2. **Очистить DNS кеш на клиенте (в Грузии)**
   ```bash
   # macOS
   sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder

   # Windows
   ipconfig /flushdns

   # Linux
   sudo systemd-resolve --flush-caches
   ```

3. **Проверить /etc/hosts файл**
   ```bash
   cat /etc/hosts | grep pylandschool
   # Удалить любые записи с pylandschool.com
   ```

4. **Использовать VPN**
   - Подключиться через VPN с серверами вне Грузии
   - Если через VPN работает → проблема в грузинском ISP

5. **Использовать альтернативный DNS**
   - Cloudflare: 1.1.1.1
   - Google: 8.8.8.8
   - OpenDNS: 208.67.222.222

6. **Проверить браузерные расширения**
   - Отключить все расширения
   - Попробовать в режиме инкогнито/приватном

7. **HTTPS Everywhere**
   - Убедиться что используется HTTPS (не HTTP)
   - HTTP может быть перехвачен провайдером

### Долгосрочные решения:

1. **Настроить Cloudflare CDN**
   - Защита от MitM атак
   - DDoS protection
   - Скрыть реальный IP сервера

2. **HSTS Preload**
   - Добавить домен в HSTS preload list
   - Браузеры будут форсировать HTTPS

3. **CAA записи**
   - Добавить CAA DNS записи
   - Запретить выпуск сертификатов не Let's Encrypt

4. **Мониторинг DNS**
   - Настроить алерты на изменение DNS записей
   - Использовать DNS мониторинг (dnsspy.io, etc.)

5. **Связаться с регистратором**
   - Включить 2FA на аккаунте PublicDomainRegistry
   - Включить Registry Lock (защита от переноса)
